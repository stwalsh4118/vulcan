## Requirements

1. Initialize Next.js 14+ project in `web/` using `pnpm create next-app` with App Router and TypeScript.
2. Configure Tailwind CSS with a dark-themed color palette (dark backgrounds, light text — terminal/infra aesthetic).
3. Set up `tsconfig.json` with strict mode and path aliases (`@/` for `src/`).
4. Configure `next.config.ts` with API proxy rewrites to the Go backend (`/api/v1/*` → `http://localhost:8080/v1/*`).
5. Create a minimal root page that renders with the dark theme to confirm setup.
6. `pnpm dev` in the `web/` directory must start the dev server without errors.

## Implementation Plan

1. Run `pnpm create next-app web --typescript --tailwind --eslint --app --src-dir --import-alias "@/*"` from project root.
2. Configure `tailwind.config.ts` with `darkMode: 'class'` and custom dark color palette.
3. Update `globals.css` with dark theme base styles (dark background, light foreground).
4. Add API proxy rewrite in `next.config.ts`: `/api/v1/:path*` → `http://localhost:8080/v1/:path*`.
5. Create minimal root page (`src/app/page.tsx`) with a "Vulcan" heading to verify rendering.
6. Add `web/` entries to root `.gitignore` if needed (`.next/`, `node_modules/`).
7. Verify `pnpm dev` starts and page loads at `http://localhost:3000`.
